Overview

Set up your own server at home using whatever hardware you already have. We'll cover hardware choices, get Docker running, and finish with a self-hosted Home Assistant install you can use to automate your apartment.

Dedicated hardware

Do you have an old laptop, mini-PC or PC tower that isn't being used? Just use it as your new server. You have the "choice" between two options:

TypeOld hardwareNew hardware
CPU architecturearmx86_64
CPU cores< 44+
RAM< 2GB2GB+
Drive< 64GB64GB+

If your hardware falls in the lower category, see the Old Hardware (Raspberry Pi) sub-page. Otherwise, New Hardware walks through the Proxmox install. Existing OS ← that's a broken wikilink — the target file doesn't exist yet, so it renders muted+strikethrough but the page still builds.

Only a laptop

No dedicated hardware? No problem. Just use your laptop. Installing Docker on existing Linux / macOS / Windows is covered in the next section.

·· 01 OS ··

This section walks through picking and installing an OS suited to running containers 24/7. The OS choice barely matters if you're going Docker-only — pick whatever you're comfortable with.

·· 02 Installing Docker ··

Docker allows you to install containers that contain an operating system and a service / program. Containers are more secure than installing a service directly on your OS. They also fix the "works on my machine", which you may be familiar with :)

Linux (Debian)

Unfortunately, Docker is not available on Debian out of the box. Install it by running (as root):

apt update                            # update packages
apt install -yqq curl                 # install curl
curl -sSL https://get.docker.com/ | bash

macOS

Docker is available via Docker Desktop on macOS, but the performance is quite bad. Instead, you can use OrbStack, which uses a different engine, but is completely compatible otherwise.

Windows

Windows is a horrible choice to use as a server, this guide is only for testing! You are required to have WSL set up already. Easiest install is Docker Desktop.

·· 03 First service ··

Now that we are all on the same page we can install our first service. HomeAssistant is basically the all-in-one smart-home solution. You can use it to control and automate anything "smart" in your household — and a lot of stuff that isn't — via your phone or a web UI.

Setting it up

Start with creating a compose.yml file in a folder you want to store your service configs. It is recommended to create a new folder for each compose.yml (one per service).

Example: HOME/Services/homeassistant/compose.yml

services:
  homeassistant:
    container_name: home_assistant
    image: "ghcr.io/home-assistant/home-assistant:stable"
    volumes:
      - ~/Services/homeassistant/data/config:/config
      - /etc/localtime:/etc/localtime:ro
      - /run/dbus:/run/dbus:ro
    restart: unless-stopped
    privileged: true
    network_mode: host

Starting HomeAssistant

To start downloading and running HomeAssistant just run (in the same folder as the compose.yml file):

docker compose up -d
  1. Check if the image is already downloaded; if not, download it.
  2. Start the service in the background (-d = detached).
  3. Open http://localhost:8123 in your browser to set up HomeAssistant.
